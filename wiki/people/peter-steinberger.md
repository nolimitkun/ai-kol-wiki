# Peter Steinberger

奥地利开发者，**OpenClaw** 作者——2026 年初爆红的开源个人 AI agent（GitHub 历史涨星最快仓库，180k+ stars）。此前创办 PSPDFKit 并运营 13 年（PDF SDK，装机量 10 亿设备），出售后 burnout，消失三年"补课人生"，2025 年因 agentic coding 重新爱上编程。2026-02 访谈时正在 Meta 与 OpenAI 之间抉择加盟（条件：OpenClaw 保持开源）。

## 背景与风格

- 一人主导开发（2026-01 单月 6600+ commits），并行 4–10 个 agent、语音输入为主；主力工具 Codex，自称"最大的无薪 Codex 广告"（[OpenClaw 访谈](../videos/20260212-lex-openclaw-steinberger.md) 01:13–01:16、02:29）。
- 反商业化直觉：项目每月倒贴 1–2 万美元，赞助全转给依赖库作者；拒绝 VC（"每个大 VC 都在我收件箱里"）与创业路线（02:18–02:22）。
- 哲学：为体验而活（好坏体验都是收获）、金钱边际效用递减、"burnout 不是工作太多而是人的冲突"（02:10–02:17）。

## 核心观点

- **玩是最好的学习方式**；OpenClaw 赢在"别人都太把自己当回事"（[访谈](../videos/20260212-lex-openclaw-steinberger.md) 00:21、02:00–02:02）。
- **Vibe coding 是蔑称**，他做的是 agentic engineering；agentic 熟练度曲线最终回归短 prompt + 对 agent 的共情（01:04–01:07）。
- **MCP 应让位于 CLI + skills**：CLI 可组合（jq 过滤）、零 context 污染，MCP 默认塞爆 context（02:38–02:42）。
- **个人 agent 是下一个操作系统**，chat 界面只是"电视上播广播"的过渡形态；80% 的 app 将消失，"每个 app 都是一个很慢的 API，不管愿不愿意"（01:48–01:51、02:43–02:57）。
- **编程会变成织毛衣**：因喜欢而做而非必要；但"你不只是程序员，你是 builder"（03:01–03:05）。
- Opus vs Codex："Opus 有点太美国，Codex 是德国人"；差异在 post-training 目标而非原始智能（01:39–01:44）。
